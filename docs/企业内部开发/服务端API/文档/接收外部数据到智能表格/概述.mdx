## 功能概要
「接收外部数据」是智能表格提供的一项开放能力，用于让外部系统通过 Webhook 协议，将数据写到智能表格中。当用户在智能表格中开启「接收外部数据」后，系统会为该表格生成唯一的 Webhook 地址，可通过标准 HTTP POST 请求新增智能表格记录。
 
典型应用场景
| 数据来源 | 应用场景|
| ------ | ------ |
| ERP、工单、订单等系统数据 | 同步数据到企微，自动化提醒跟进 |
| 智能机器人、Dify 等工作流产生数据 | 智能表格作为数据库，可视化展示数据|
| HR、财务等系统数据 | 自动写入到智能表格查看、分析 |
 
## 「接收外部数据」配置说明

入口一：智能表格文档右上角菜单 - 点击「接收外部数据」- 开始配置 - 选择目标工作表

<img src="https://wework.qpic.cn/wwpic3az/449094_KfleKuCDTQCLDpk_1769174859/0" width=320/>

入口二：工作表三个点菜单 - 点击「接收外部数据」- 开始配置
 
 
在目标智能表格的工作表中启用「接收外部数据」功能，复制目标工作表的 Webhook 地址，通过该 Webhook 地址按照对应「示例数据」的格式传入数据，即可实现向该工作表添加新记录。
 
权限说明
- 拥有 Webhook 地址的任何方均可向该工作表新增记录，一定要<font color="red">**保护好接收外部数据的Webhook地址**</font>，避免泄漏！不要分享到github、博客等可被公开查阅的地方，否则坏人就可以用你的Webhook地址向工作表中写入垃圾内容。
- 若发现Webhook 地址泄露，请在智能表格「接收外部数据」设置中立即停用对应工作表的Webhook地址并重新配置新的Webhook地址。
- 支持配置推送来源 IP 白名单。启用后，仅允许在白名单内的IP向工作表新增记录。
- 智能表格支持通过 Webhook 更新已有记录的数据。为保障数据安全与可控性，当前仅允许更新由 Webhook 写入的记录，不支持更新人工创建或通过其他方式导入的记录。
 
 
## 如何使用「接收外部数据」
开发者开启「接收外部数据」获取到对应工作表Webhook地址后，可以为智能表格工作表[添加记录](#60594)和[更新记录](60595)。
 
以添加记录为例，按以下说明向Webhook地址发起HTTP POST 请求，即可实现为该智能表格工作表添加记录
假设对应工作表的Webhook地址是：https://qyapi.weixin.qq.com/cgi-bin/wedoc/smartsheet/webhook/add_records?key=QIikPDZRKD7tw2m4AdZjZ4aqcfzxeqwGKL3USxhZ9yehr3X92jcK3zhl2t3PrcZgxxxxxxxxxxxxxxxxxx
 
以下是用curl工具往智能表格工作表添加记录的示例：
<font color="red">**注意要将url替换成你的工作表Webhook地址，请求的 json 示例由 接收外部数据 - 示例数据中获取，content必须是utf8编码**</font>
 
```
curl 'https://qyapi.weixin.qq.com/cgi-bin/wedoc/smartsheet/webhook/add_records?key=QIikPDZRKD7tw2m4AdZjZ4aqcfzxeqwGKL3USxhZ9yehr3X92jcK3zhl2t3PrcZgxxxxxxxxxxxxxxxxxx' \
   -H 'Content-Type: application/json' \
   -d '
   {
        "add_records": [
            {
                "values": {
                    "FIELD_ID": [
                        {
                            "type": "text",
                            "text": "文本内容"
                        }
                    ]
                }
            }
        ]
    }'
```
 
## 频率限制
每个工作表的Webhook累计添加（更新）记录不能超过3000条/分钟
每个智能表格文档的所有 Webhook 累计添加(更新)记录不能超过 10000 条/分钟
