用于获取指定智能文档中的页面层级结构信息。
 
**请求方式**：POST（**HTTPS**）
**请求地址**：https://qyapi.weixin.qq.com/cgi-bin/wedoc/smartdoc/get_page_hierarchy?access_token=ACCESS_TOKEN
 
**请求包体**
```json
{
    "docid": "DOC_ID"
}
```

**参数说明** 
| 参数 | 类型 | 是否必须 | 说明 |
|------|------|----------|------|
| docid | string | 是 | 智能文档的文档 id，可通过 [新建文档](#43939) 接口创建 |
 
**权限说明**
- 自建应用需配置到"可调用应用"列表中的应用secret所获取的accesstoken来调用
- 第三方应用需具有"文档"权限
- 代开发自建应用需具有"文档"权限

**返回示例**
 
```json
{
    "errcode": 0,
    "errmsg": "ok",
    "pages": [
        {
            "page_id": "PAGE_ID_1",
            "parent_id": "",
            "title": "首页"
        },
        {
            "page_id": "PAGE_ID_2",
            "parent_id": "PAGE_ID_1",
            "title": "子页面1"
        },
        {
            "page_id": "PAGE_ID_3",
            "parent_id": "PAGE_ID_1",
            "title": "子页面2"
        }
    ]
}
```
 
**参数说明**
| 参数 | 类型 | 说明 |
|------|------|------|
| errcode | int32 | 错误码 |
| errmsg | string | 错误码说明 |
| pages | object[] | 页面层级列表 |
| pages[].page_id | string | 页面 ID |
| pages[].parent_id | string | 父页面 ID，为空表示是根页面 |
| pages[].title | string | 页面标题 |

### 页面层级关系说明
通过 `parent_id` 字段可以构建页面的树形层级结构
```
首页 (parent_id = "")
├── 子页面1 (parent_id = "首页的page_id")
│   ├── 孙页面1 (parent_id = "子页面1的page_id")
│   └── 孙页面2 (parent_id = "子页面1的page_id")
└── 子页面2 (parent_id = "首页的page_id")
```
