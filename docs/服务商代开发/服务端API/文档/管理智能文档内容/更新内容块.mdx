用于批量更新指定页面中已有的 Block。
 
**请求方式**：POST（**HTTPS**）
**请求地址**：https://qyapi.weixin.qq.com/cgi-bin/wedoc/smartdoc/update_blocks?access_token=ACCESS_TOKEN
 
**请求包体**
```json
{
    "docid": "DOC_ID",
    "page_id": "PAGE_ID",
    "blocks": [
        {
            "id": "BLOCK_ID",
            "type": "BLOCK_TYPE_TEXT",
            "content": "Updated Content",
            "props": {}
        }
    ]
}
```
 
**请求参数说明**
| 参数 | 类型 | 是否必须 | 说明 |
|------|------|----------|------|
| docid | string | 是 | 智能文档的文档 id |
| page_id | string | 是 | 页面的id |
| blocks | object\[\]([BlockInfo](#61643/blockinfo)) | 是 | 需要更新的 Block 列表 |
| blocks[].id | string | 是 | 需要更新的 Block ID |
| blocks[].type | string | 否 | Block 类型，参见 [BlockType](#61643/blocktype) |
| blocks[].title | string | 否 | Block 的文本内容 |
| blocks[].parent_id | string | 否 | 父节点 Block ID |
| blocks[].after_id | string | 否 | 插入位置，表示排在哪个 Block 之后，为空表示插入到最前 |
| blocks[].children | string[] | 否 | 子节点 ID 列表 |
| blocks[].props | BlockProps | 否 | Block 属性 |

**权限说明**
- 自建应用需配置到"可调用应用"列表中的应用secret所获取的accesstoken来调用
- 第三方应用需具有"文档"权限
- 代开发自建应用需具有"文档"权限

**返回示例**
```json
{
    "errcode": 0,
    "errmsg": "ok",
    "blocks": [
        {
            "id": "BLOCK_ID",
            "type": "BLOCK_TYPE_TEXT",
            "title": "Updated Content",
            "parent_id": "PARENT_BLOCK_ID",
            "children": [],
            "props": {}
        }
    ]
}
```
 
**返回参数说明**
| 参数 | 类型 | 说明 |
|------|------|------|
| errcode | int32 | 错误码 |
| errmsg | string | 错误码说明 |
| blocks | object\[\]([BlockInfo](#61643/blockinfo)) | 更新成功后返回的 Block 列表 |
