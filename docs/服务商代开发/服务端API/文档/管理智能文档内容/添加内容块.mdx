用于向指定页面中批量添加内容块。
 
**请求方式**：POST（**HTTPS**）
**请求地址**：https://qyapi.weixin.qq.com/cgi-bin/wedoc/smartdoc/add_blocks?access_token=ACCESS_TOKEN
 
**请求包体**
 
```json
{
    "docid": "DOC_ID",
    "page_id": "PAGE_ID",
    "blocks": [
        {
            "type": "BLOCK_TYPE_TEXT",
            "title": "Hello World",
            "parent_id": "PARENT_BLOCK_ID",
            "after_id": "AFTER_BLOCK_ID",
            "props": {}
        }
    ]
}
```

**参数说明**
 
| 参数 | 类型 | 是否必须 | 说明 |
|------|------|----------|------|
| docid | string | 是 | 智能文档的文档 id，可通过 [新建文档](#43939) 接口创建 |
| page_id | string | 是 | 页面的id |
| blocks | object\[\]([BlockInfo](#blockinfo)) | 是 | 需要添加的 Block 列表 |
| blocks[].type | string | 是 | Block 类型，参见 [BlockType](#blocktype) |
| blocks[].title | string | 否 | Block 的文本内容 |
| blocks[].parent_id | string | 否 | 父节点 Block ID |
| blocks[].after_id | string | 否 | 插入位置，表示排在哪个 Block 之后，为空表示插入到最前 |
| blocks[].children | string[] | 否 | 子节点 ID 列表 |
| blocks[].props | object | 否 | Block 属性 |

 
**权限说明**
- 自建应用需配置到"可调用应用"列表中的应用secret所获取的accesstoken来调用
- 第三方应用需具有"文档"权限
- 代开发自建应用需具有"文档"权限

**返回示例**
```json
{
    "errcode": 0,
    "errmsg": "ok",
    "blocks": [
        {
            "id": "BLOCK_ID",
            "type": "BLOCK_TYPE_TEXT",
            "title": "Hello World",
            "parent_id": "PARENT_BLOCK_ID",
            "after_id": "AFTER_BLOCK_ID",
            "children": [],
            "props": {}
        }
    ]
}
```
 
**返回参数说明**
| 参数 | 类型 | 说明 |
|------|------|------|
| errcode | int32 | 错误码 |
| errmsg | string | 错误码说明 |
| blocks | object\[\]([BlockInfo](#blockinfo)) | 添加成功后返回的 Block 列表（含生成的 ID） |
