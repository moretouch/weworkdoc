## 背景说明
为提升企业数据安全，2026年6月15日起，企业微信将对以下两类数据启用「定期授权」机制：
- 获客助手接口数据：即广告投放时的转化数据，包括首次开口、多次开口数据
- 数据与智能专区接口数据：即通过数据专区接口可获取的相关数据

启用定期授权后，企业管理员单次授权上述数据的有效期为 90 天，到期需重新授权方可继续调用接口获取数据。

## 定期授权机制介绍
###1. 首次授权
应用安装完成后，若应用有申请需定期授权的数据权限，将通过应用消息向安装人（管理员）推送权限授权消息，管理员点击消息即可完成授权。
![](https://wework.qpic.cn/wwpic3az/535611_LO6HsdTsTbSuQ0z_1779865699/0)

###2. 有效期
授权完成后 90 天有效，到期后将无法通过接口继续返回数据，必须由企业管理员重新授权。
6月15日及之后新授权企业，自授权日起算90天；6月15日之前已授权企业，自6月15日统一起算90天。（说明：历史已授权的外部服务应用、灰度数据专区文档权限的应用自数据授权日起算 90 天，如有疑问可咨询企业微信产品客服。）

###3. 到期续期提醒
企业微信将在数据权限到期前 7 天/3 天/1 天和到期当天，通过应用消息向管理员推送消息提醒，管理员点击消息即可重新授权。授权后，有效期从授权日起重新起算 90 天。
![](https://wework.qpic.cn/wwpic3az/377239_l890ImdIRPOwqzO_1779866351/0)

###4. 管理入口
企业管理员可在「应用详情管理页-权限详情页」主动管理数据的授权，包括进行数据授权、修改授权数据范围、取消数据授权等操作。
![](https://wework.qpic.cn/wwpic3az/379168_EwGhn9gpQU2FMrm_1779866542/0)

###5. 通过JSAPI引导授权
企业微信提供了JSAPI/组件，开发者可通过调用JSAPI/组件打开授权页（包括首次授权、续期授权），引导企业进行授权。
调用 JSAPI/组件 打开授权页时，若未授权该权限点击同意授权按钮完成权限授权，若已授权该权限点击同意授权按钮完成权限续期。
说明：JSAPI（[查看文档](https://developer.work.weixin.qq.com/document/path/95528 "查看文档")） 用于应用在企业微信内使用场景调用，组件（[查看文档](https://developer.work.weixin.qq.com/document/path/101462 "查看文档")）用于外部浏览器场景调用。

## 服务商应用接入
服务商可于2026年6月15日启用「定期授权」机制后开始接入。
###1. 使用「获客助手接口」的应用
a）在服务商助手「工具-应用开发-应用详情页」填写数据用途描述与使用场景，须如实填写，内容将在授权时展示给企业管理员（必须填写，2026年6月15日起支持填写）

b）通过 API 获取数据权限的到期时间 [查看文档](https://developer.work.weixin.qq.com/document/path/99052)
c）在应用内接入授权 JSAPI，引导企业管理员完成授权与续期 [查看文档](https://developer.work.weixin.qq.com/document/path/95528 "查看文档")

###2. 使用「数据与智能专区接口」的应用
a）在「数据与智能专区调用」的程序详情页，设置程序可输出的数据范围，请按实际调用的接口如实设置，将限定程序可调用的接口范围（必须设置，当前即可开始设置）

## 常见问题
**Q1: 2026年6月15日上线后，服务商应用还未调整，会立即导致不可用吗？**
不会。 6.15 上线定期授权机制后：1）新授权企业：自授权日起 90 天有效，到期未续期则接口不返回数据；2）历史已授权企业：以 6.15 为起始日重新计算 90 天，即 最早可能到期日为 9.13 前后。
这意味着服务商有 约 90 天的缓冲期 完成产品调整、客户触达、续期 SOP 落地等；在此之前不会导致服务商应用不可用。

**Q2：数据用途描述和场景填写是怎么维护的？一个应用维护一次还是每个客户都要维护一次？**
按照“应用”维度维护的。用途描述和场景截图会在企业授权数据时展示给企业，建议用清晰、面向客户视角的描述，以提升企业的授权成功率。

**Q3：企业如何授权数据权限，可以通过哪些入口进行授权？**
支持通过以下 3 种入口进行授权：
1)应用消息推送：到期前 7 天 / 3 天 / 1 天和到期当天，企业微信通过应用消息向管理员推送续期提醒卡片，管理员点击进行授权；
2)应用管理详情页：管理员可在「企业微信管理后台 - 应用管理 - 对应应用详情页」主动管理授权状态；
3)服务商应用内（推荐）：服务商可接入 授权 JSAPI，在自己的应用界面提供授权入口，让管理员无需跳出即可完成授权。

**Q4: 服务商可以在应用界面自己做授权入口、展示到期时间吗？**
完全支持，且这是推荐做法。

**Q5：服务商可以提前引导客户进行续期吗？**
可以，且这是推荐做法。除官方 7/3/1 天的应用消息提醒外，服务商可以：通过 API / 回调获取每家企业的到期时间，自行设置提醒（例如 30 天 / 14 天 / 7 天），并通过 JSAPI 引导企业进行重新授权。

**Q6：90 天有效期会影响存档"最长 2 年"的规则吗？**
两者是不同维度，无相互影响。90天有效期管理的是应用的数据获取权限，2 年存档管理的是消息最长可存储的时长。

**Q7：数据专区数据授权到期后，消息会继续存档吗？**
数据专区会话数据授权到期不影响消息继续存档，即数据专区会话数据授权到期后，新产生消息也会继续存档。

**Q8：数据授权到期，会影响哪些API？**
请参考正文的「授权到期影响」部分。
