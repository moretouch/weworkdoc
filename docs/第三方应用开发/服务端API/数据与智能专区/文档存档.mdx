## 概述

企业微信支持应用在数据与智能专区中，检索企业授权的文档。目前该功能仅灰度开放，已灰度服务商为应用选择对应权限后，即可为企业提供文档检索服务。

## 应用准备
### 1.服务商创建应用
勾选“分析企业文档数据”权限，并选择对应的分析工具。
![enter image description here](https://wework.qpic.cn/wwpic3az/627559_rwehiwHCRvCwfFp_1777467904/0)

### 2.企业授权应用及数据权限
企业用户扫码安装应用后，点击应用发送的卡片消息，在跳转的页面授权数据权限：

- 企业管理员：选择文档成员范围配置数据，完成授权。
- 数据范围：被授权成员有浏览权限的、浏览人数 ≥ 2 的公开文档会进入专区。
- 授权机制：数据有效期90天，到期后企业管理员需重新授权。

![enter image description here](https://wework.qpic.cn/wwpic3az/786241_ucyPwEuAScuKWKh_1777530153/0)

如果企业尚未开启文档存档功能，参见**附录**。

## 如何调用
### 1. oauth2授权
在数据专区检索文档，需要用户通过oauth2授权进入应用，若企业管理员已为应用开启了数据与智能专区文档存档权限，则应用调用[获取访问用户身份](#15122)时，企业微信返回user_doc_ticket，可在30天内调用数据专区通用模型时检索该用户的文档。

### 2. 调用通用模型能力
服务商应配置通用模型能力：
![enter image description here](https://wework.qpic.cn/wwpic3az/488341_8lX6sj2vTnqQAt5_1777469597/0)
其中，{doc_fragments}变量用于替换检索文档之后的片段数据。

接下来，应用在调用[通用模型能力接口](#56425)时，传入doc_retrieval_params参数（包括关键词），企业微信数据专区使用关键词对用户有权限的文档进行检索，再将检索得到的片段替换上述输入协议的{doc_fragments}变量，然后调用通用大模型。

### 3. 成员选择专区文档
在[通用模型能力接口](#56425)中，应用在doc_retrieval_params参数还可指定docid列表，以指定文档检索范围。应用可通过jsapi[数据与智能专区文档选择](#61186)或小程序接口[数据与智能专区文档选择](#61183)，让用户选择其创建的doc文档，应用即可获得相应文档的docid列表。
